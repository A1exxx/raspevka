## 1. Состояние сейчас (готово ✓)

- PWA: manifest + sw.js (network-first HTML, cache-first ассеты) + иконки 192/512/maskable/apple-touch.
- Версия-баннер «Доступна новая версия» (version.json штампуется при сборке).
- Страница сверки нот /sverka.html, демо-виджет /demo.html.
- Тесты: **162/162** (`npm test`). E2E: `node tools/e2e-smoke.mjs` (зелёный).

## 3. Усиливающие улучшения (не глобальные — «дожимающие»)

### Воронка и рост (приоритет — прямо влияют на лиды)
- **OG-карточка + favicon** для /raspevka (см. §2.3) — репост ссылки выглядит дорого.
- **«Поделиться результатом»** после распевки (звёзды + точность картинкой) — виральная петля.
- **UTM-метки** на ссылках сайт↔приложение — видно, какой канал приводит.
- **A2HS-подсказка** «Добавь на главный экран» (PWA install prompt) — возвраты ↑.
- **Сведение лидов в одну таблицу** (сайт + приложение шлют в один Telegram/Notion) —
  менеджеру не нужно смотреть в два места.

### Удержание (после первого визита)
- **Пуш-напоминание о стрике** (PWA Notifications, по согласию) — «не теряй серию».
- **Звук успеха на 3★** — короткий приятный аккорд (НЕ синтез ffmpeg — взять из звуковой
  библиотеки, по правилу no-synth-audio).
- **Шаг транспозиции в программе** — у баритона распевка = до 25 повторов (~10 мин);
  кнопка «Закончить на повторе» уже есть, но дефолтный шаг 2 полутона сделает короче
  (согласовать с музыкантом — меняет методику).

### Доверие и качество
- **Скриншоты для будущих сторов** (RuStore/Play) — 5–6 кадров ключевых экранов.
- **Пустые/ошибочные состояния** микрофона (запрет доступа, нет голоса) — проверить копирайт.
- **Аналитика «почему ошибка»** (flat/sharp/зажим → совет) — главный дифференциатор vs Vocalista, частично есть в разборе, можно усилить трендом между сессиями.

### Гигиена
- Старые MP3-дубли в `docs/note-previews` (имена до переименования) — удалить по команде.
- `tools/_*.{jpg,mjs}` — временные артефакты прогонов, теперь в .gitignore.
