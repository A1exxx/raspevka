# Распевка — продуктовый план (этап разработки)

_Составлено на основе ресёрча конкурентов 2025–2026. Это рабочий план «что и как делать», не финал._

---

## 1. Монетизация и триал

**Что делают конкуренты (факты):**
- Yousician / Simply Sing — hard paywall, **7-дневный триал только на годовом плане**, ~$60–90/год. Simply Sing так делает **~$1M/мес**.
- Singing Carrots / Riyaz / Vanido — freemium с лимитом (Vanido — 3 упр./день, Riyaz — 8 мин/день).
- Duolingo с июля 2025 ввёл energy-cap как монетизацию → **массовый бунт аудитории** (антипаттерн).
- Ключ: **«продажа происходит ДО пейволла»** — magic-demo в онбординге (ИИ слушает голос → «вот твой диапазон»), быстрый успех, эмоция, и только потом подписка.

**Рекомендуемая модель для нас (приложение ведёт в школу + подписка):**
1. **Freemium + magic-demo онбординг** — анализ голоса/диапазона как «вау» бесплатно (у нас уже есть тест голоса — усилить celebration).
2. **Soft-paywall после 3–5 распевок** (count-cap, НЕ time/energy — те бесят).
3. **7-дневный триал на годовой подписке**.
4. **Отдельный путь в школу** — «Хочешь с живым педагогом? → школа» без пейволла (перелив трафика из инвест-модели).
5. **A/Б:** триал на всех планах vs только годовой; 3 распевки/день vs 10 мин/день.
6. **НЕ делать** жёсткий energy-cap как монетизацию — вокальная аудитория «нежнее» языковой.

## 2. Тест-режим / QA-панель (запрос пользователя)

Цель: тестировать триал/стрик/энергию/пейволл, «перематывая время», и проверять путь реального пользователя.

**Архитектурная основа:** централизовать доступ к времени — везде в коде вместо `Date.now()` ходить через `clock.now()`. Тогда в тест-режиме можно подменить «сегодня» и проверить:
- истечение 7-дневного триала,
- набор/потерю стрика и его заморозку,
- восстановление энергии,
- открытие блоков по дням.

**Два режима:**
- **Внутренний (dev):** скрытая панель (вход `?dev=1` или долгий тап по логотипу). Кнопки: сдвинуть дату ±дни, задать стрик/энергию/день триала/тариф/сданные экзамены, разблокировать всё, сброс.
- **Пользовательский (preview):** «прожить» реальный путь — запустить 7-дневный триал, увидеть как появляется пейволл, как ведёт себя напоминание. Без читов — чистая симуляция.

**Сложность:** низкая-средняя. Главное — ввести `clock` и `trialState` в `progress.js`.

## 3. Геймификация и удержание (retention)

**Факты:** стрик у Duolingo снизил чёрн 47%→28%, DAU +36%. Streak Freeze усиливает «владение». D30 edtech ~12%. Антипаттерны: hard-cap как монетизация, grinding ради стрика, >1–2 пуша/день.

**Уже есть:** стрик, энергия, программа-блоки, экзамены, скоринг, «фокус дня».

**Добавить — Приоритет 1 (бьёт по D7):**
- **Заморозка стрика** (streak freeze) — мало кода, сильный эффект.
- **Ежедневная цель + напоминание** (локальное; PWA Push ограничен на iOS).
- **«Распевка дня» как ритуал** (усилить текущий «фокус дня» → одна чёткая задача).
- **Celebration при завершении** — pulse/конфетти + бейдж-веха.

**Приоритет 2 (D30):**
- Достижения/бейджи («Первая октава», «7 дней подряд», «Чистое вибрато»).
- Видимый прогресс к следующему блоку на главной (частично есть).

**Избегать:** time/energy-cap как монетизация, агрессивные пуши.

## 4. Музыка (по ТЗ музыканта)

**Сделано:** распевки на гласных из PDF (Cheryl Porter L02), система ладов (Free/Standard/Pro), скоринг вибрато (ровность/частота), грув per-exercise (8 характеров), фонограмма «пой под запись», дыхательные, скачки/гаммы.

**Осталось:**
- Наполнить блоки 3–25 нотами музыканта (ждём ноты/MIDI/PDF).
- Уникальная грув-мелодия под каждую распевку (сейчас адаптивный паттерн).
- Расширить вибрато-упражнения (kickstart / стабильность / фразы — по ТЗ).
- Синхронизация нотного хайвея с фонограммой (нужны ноты трека).
- Блоки 22–25 (аккорды/ансамбль) — запись+наложение голоса (фундамент записи уже есть).

## 5. Дизайн — премиально и современно (тренды 2026)

**Факты:** премиальные музыкальные приложения 2026 — **тёмная база** (near-black), **luminous pitch trail с glow**, цветовые зоны точности (плавный hue, не мигание), glassmorphism для HUD, **крупная моноширинная цифра** реального времени, структурная микромоция + haptics на попадании. Дешёвый UI — светлый фон, кислотные цвета, мигание «мимо».

**Решение (разобрал варианты):** не делать всё приложение тёмным (дружелюбная светлая оболочка хорошо стыкуется со школой), а **игровой/нотный экран сделать тёмным** — там тёмный фон + светящийся след высоты выглядят дорого. Лучшее из двух.

**5 улучшений именно для real-time pitch UI:**
1. Pitch-trail со свечением и тающим хвостом; на точном попадании — короткая вспышка (glow pulse 200ms).
2. Цветовые зоны точности: зелёный (±20ц), жёлтый (±50ц), красный — плавный hue-shift, без мигания.
3. Тёмный фон хайвея + ноты как слегка светящиеся пластины; целевая нота «оживает» при приближении.
4. Крупная цифра по центру: нота + Гц + центы, моноширинный шрифт — читается краем глаза при пении.
5. Микромоция при попадании: scale-pulse 1.0→1.05 + haptic (`navigator.vibrate`), без конфетти.

(Светлая оболочка уже прокачана: Fraunces-заголовки, тинт-тени, тактильность.)

## 6. Демо-кусок для сайта школы (встраиваемый «вау»-виджет)

**Идея:** один интерактив на лендинге «Прояви» — «нажми микрофон, спой ноту, увидь свою высоту». Редкий дифференциатор (так делают Singing Carrots, Spardha School of Music — большинство школ нет).

**Архитектура (продумать сразу):**
- Вынести ядро (mic.js + pitch-detector.js + минимальная визуализация одной распевки) в **отдельную точку входа** `demo.html` + `src/demo.js`, БЕЗ зависимости от полного приложения.
- Vite multi-entry → отдельный маленький бандл. На сайт школы встраивается **одной строкой**: `<iframe src="https://a1exxx.github.io/raspevka/demo.html">` + кнопка «Открыть полное приложение».
- Ядро общее с приложением → обновляешь приложение, демо обновляется само.
- Демо = самое простое и эффектное: «спой и увидь ноту» или одна короткая распевка с попаданием в цвет.

## 7. Предлагаемая последовательность сборки

1. **Фундамент тестирования:** `clock`-абстракция + QA-панель (dev/preview). Без неё нельзя нормально тестить триал/удержание.
2. **Retention P1:** заморозка стрика + ежедневная цель + celebration.
3. **Премиум pitch-экран:** тёмный игровой экран + glow-trail + зоны + крупная цифра + haptic.
4. **Монетизация:** логика триала/пейволла (сначала за тест-режимом, потом A/Б).
5. **Демо-виджет** для сайта школы.
6. **Потом** — обсуждение публикации в сторы (Telegram-воронка → Google Play → RuStore → Apple).
