# Заявки «Урок с педагогом» → Telegram за 2 минуты

Сейчас заявки сохраняются локально на устройстве ученика. Чтобы они мгновенно
прилетали педагогу в Telegram — нужен бот (бесплатно, 2 минуты):

## Шаг 1. Создать бота
1. В Telegram открой **@BotFather** → команда `/newbot`.
2. Имя: например `Распевка — заявки`. Username: например `raspevka_leads_bot`.
3. BotFather пришлёт **токен** вида `12345678:AAE...` — скопируй.

## Шаг 2. Узнать chat id
1. Напиши СВОЕМУ новому боту любое сообщение (например «старт») — от того
   Telegram-аккаунта, куда должны приходить заявки (педагог/менеджер).
2. Открой в браузере (подставь токен):
   `https://api.telegram.org/bot<ТОКЕН>/getUpdates`
3. В ответе найди `"chat":{"id":987654321` — это **chat id**.

## Шаг 3. Вписать в приложение
Открой `src/state/lead-config.js` и заполни:
```js
export const LEAD_TG = {
  token: '12345678:AAE...',
  chatId: '987654321',
};
```

## Шаг 4. Пересобрать и задеплоить
```
npm run build
# деплой dist в gh-pages (см. docs/HANDOFF.md «Деплой»)
```

Готово. Каждая заявка приходит сообщением:
имя, контакт, предпочтение педагога, цель + прогресс ученика (тип голоса,
диапазон, стрик, блоки) — педагогу сразу видно, с чего начинать.

⚠️ Токен в клиентском коде виден любому, кто откроет исходники страницы.
Для лид-бота это приемлемый компромисс (бот умеет только слать сообщения в один чат),
но не используй этот токен ни для чего другого. Если станет проблемой — следующий
шаг: бесплатный Cloudflare Worker как прокси (токен на сервере).
